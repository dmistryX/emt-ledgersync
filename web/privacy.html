Privacy Policy

Last Updated: April 13, 2026

1. Core Model

Emendate LedgerSync is a desktop application. It runs locally on your computer and does not use an Emendate cloud backend for data processing.

2. Data We Do Not Collect

  • Personal identity data
  • Excel file contents for external analytics
  • Telemetry, ad tracking, or usage profiling
  • IP-based analytics for product behavior tracking

3. Local Data Storage

3.1 Local SQLite Database

  • File: Emendate_LedgerSync.sqlite3
  • Location: %APPDATA%\<install-folder-name>\
  • Contains settings, encrypted credentials, company mappings, and OAuth tokens

3.2 Log File

  • File: Emendate_LedgerSync.log
  • Purpose: diagnostics and troubleshooting

3.3 Excel Files

  • Source Excel files remain on your device
  • Only selected import payloads are sent to QuickBooks APIs

4. Data Flow

  • Desktop app authenticates directly with QuickBooks OAuth
  • Desktop app sends selected bill/invoice payloads directly to QuickBooks APIs
  • No intermediary Emendate server is used in this flow

5. Third-Party Service

QuickBooks Online (Intuit) is a required third-party API provider. Data sent to QuickBooks is governed by Intuit policies and terms.

6. Security Notes

  • Credentials and tokens are stored locally and encrypted in the app database.
  • Protect your machine account, filesystem access, and backups.
  • Revoke app access from QuickBooks if needed.

7. Data Removal

To clear local app data, close the app and remove:

  • Emendate_LedgerSync.sqlite3
  • Emendate_LedgerSync.sqlite3-shm (if present)
  • Emendate_LedgerSync.sqlite3-wal (if present)

8. Changes To This Policy

We may revise this document as product behavior changes. The date above reflects the latest revision.

Summary: Emendate LedgerSync is designed as a local-first desktop tool. Your data stays on your machine except the records you explicitly send to QuickBooks.
